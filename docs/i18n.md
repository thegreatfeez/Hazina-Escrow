# Internationalization Support

The frontend now has integrated i18n support wired into the running app.

## Available Locales

- English (`en`)
- Kiswahili (`sw`)
- French (`fr`)
- Spanish (`es`)

## What Is Included

- `I18nProvider` wrapped around the app entrypoint
- Locale detection from browser preferences
- Locale persistence in `localStorage`
- Shared `useI18n()` hook for translated copy and formatting helpers
- Reusable `LocaleSwitcher` rendered in the navbar
- Typed translation catalogs with a shared message shape
- Localized UI copy across the main app routes and shared components
- Locale-aware number, currency, and relative-time formatting

## Notes

- Backend-provided research/report text is still shown as returned by the API.
- Dataset type badges are localized in the UI.
- Missing translation keys fall back to English.
